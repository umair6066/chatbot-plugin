New

Generic Product Types

The Product type is now generic — each client defines their own product shape with full TypeScript safety. Base fields are always included; extra fields are added as the generic parameter:

Step 1 — Define your type

// Your custom shape for a watch dealer
type WatchProduct = Product<{
  model:   string;
  reg:     string;    // reference number
  serial?: string;
  metal?:  string;
  dial?:   string;
  status?: string;
}>;

Step 2 — Create your products

const watches: WatchProduct[] = [
  {
    id:          '48291',
    name:        'Rolex, Submariner Date, 116610LN, 2019, Steel',
    price:       12850,
    inStock:     true,
    // Your custom fields ↓
    model:       'Submariner Date',
    reg:         '116610LN',
    metal:       'Steel',
    dial:        'Black',
    status:      'Purchased/InStock',
  },
];

Step 3 — Pass to the widget

<ChatbotWidget products={watches} />

New

API-Backed Search — onSearch

For large or live catalogs, pass an onSearch callback instead of loading products upfront. The widget calls it for every non-greeting message and renders the returned products as cards. No pre-loading — the API is only called when the user actually searches.

React usage

<ChatbotWidget
  onSearch={async (query) => {
    const res = await fetch(
      `/api/products/search?q=${encodeURIComponent(query)}&per_page=10`
    );
    const data = await res.json();
    return data.products.map(mapProduct);
  }}
/>

BW Dealer App — actual API example

async function searchInventory(query: string): Promise<WatchProduct[]> {
  const res = await fetch(
    `/admin/products?search=${encodeURIComponent(query)}`
    + `&status=Purchased/InStock,Consignment,Working On`
    + `&per_page=10&page=1`
  );
  const data = await res.json();
  return data.data.products.map((p: any) => ({
    id:       p.chatbotPluginProduct.id,
    name:     p.chatbotPluginProduct.title,
    price:    p.chatbotPluginProduct.retailPrice,
    model:    p.chatbotPluginProduct.model,
    reg:      p.chatbotPluginProduct.reg,
    dial:     p.chatbotPluginProduct.dial,
    metal:    p.chatbotPluginProduct.metal,
    status:   p.chatbotPluginProduct.status,
    inStock:  p.chatbotPluginProduct.status !== 'Working On',
  }));
}

Improvements

Improved

All bot responses rewritten to sound natural rather than mechanical:

Before

Suggestion Chips

  • Chips update after every bot reply with context-aware options
  • Older chips grey out — only the latest reply's chips are tappable
  • "Show X more" chip added automatically when results are capped
